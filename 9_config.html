Всі користувацькі настройки винесені в окремий файл config/params/param.php

Тут ви можете поміняти настройки зази даних, щаблонів і тд., а також створювати свої параметри, які можете використовувати в настройках компонентів або де інде.

Наприклад можна створити свій параметр, добавивши рядок у вищезгаданий файл

'myparam' => 1

потім наприклад можна його підставити в настройки компонента

'myComponent2' => [ 'class' => '\Component\MyComponent2', 'params' => [ 'a' => [ 'value' => '%myparam%' ], ] ],

тут у новоствореному компоненті у властивість "а" підставиться 1

Також можна параметри використовувати наступним чином

нехай маємо вищезгаданий компонент

'myComponent' => [ 'class' => '\Component\MyComponent', ],

створимо для нього параметр

'dynamicComponent' => 'myComponent'

і потім цей компонент підставимо у інший компонент через створений параметр

'myComponent2' => [ 'class' => '\Component\MyComponent2', 'params' => [ 'myComponent' => [ 'type' => 'object', 'value' => '%dynamicComponent%' ], ] ],

Тут ви бачете як через параметри можна динамічно міняти компоненти, які підставляються в інші компоненти. Це дає можливість швидко поміняти клас обробник якоїсь дії на свій клас.

Також значенням параметра може виступати масив.
