Для створення контролера, потрібно перше прописати в настройках урли, які будуть взаємодіяти з контролером. Для цього відкриєм файл config/routing/route.php.

Його структура є у вигляді масиву, який повертається (за таким принципом побудовані всі конфіги у фреймворці)

Ось приклад роута (маршрут)

'myfirst' => [ 'url' => '/my/first', 'controller' => '\Controller\My\First', 'method' => 'index', 'params' => 'a/b/c', ],

перший рядок - це назва маршруту, назва повинна бути унікальні.

url - це урл, який в браузері буде відповідати даниму маршруту.

controller - це назва класу, який буде відповідати даному маршруту. Тут видно, що контролер може бути в любій папці. Наприклад можна створити наступні папки і клас в них: a/b/c/d.php, в даному випадку controller буде \a\b\c\d (великі малі букви мають значення).

method - це метод (action), який в даному класі буде відповідати даному маршруту.

params - це дуже важлива частина маршрута. Вона відповідає за кількість та назву параметрів, які будуть передані методу контролера. Тут наведу приклад.

В даному прикладі урли у рядку браузера, які відповідають даному роуту, можуть бути:

site.loc/my/first/1/3/4

site.loc/my/first/7/8/andriy

site.loc/my/first/roman/1/and

Як ви вже зрозуміли за a/b/c будуть відповідати 1/3/4, 7/8/andriy, roman/1/and відповідно.

Якщо параметрів не потрібно цей параметр в маршруті пропускаємо

Приклад контролера під даний маршрут

<?php namespace Controller\My; use Controller\BaseController; /** * @example */ class First extends BaseController { /** * * @param string $a * @param string $b * @param string $c */ public function index($a,$b,$c) { } }

у змінні a, b, c прийдуть 1/3/4, 7/8/andriy, roman/1/and відповідно

також, коли ми можем створити наступний маршрут

'mysecond' => [ 'url' => '/my/first', 'controller' => '\Controller\My\First', 'method' => 'index1', 'params' => 'a', ],

цьому маршруту буде відповідати метод і він буде розцінюватися як зовсім інший

/** * * @param string $a */ public function index1($a) { }

ще в настройках маршрута існує параметр role, але про нього поговорим пізніше.

З прикладу ви зауважили що всі контролери наслідують BaseController, це зроблено для того щоб в BaseController провести перщочергові настройки і ініціалізацію змінних, які будуть доступні у ваших контролерах.

Контролкр можна не наслідувати, але це можна робити тоді, коли ви точно знаєте що ви робите і це буде ефективно..
